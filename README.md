# PersonOS

PersonOS 是一个长期沉淀个人知识资产与判断方式的仓库。当前阶段只建设可读、可维护、可追溯的知识结构，不训练模型，不接入向量库、Agent、数据库或后端服务。

## 目标

- 记录个人画像、能力边界、偏好与反模式。
- 沉淀项目经验、决策过程、失败复盘和成功模式。
- 建立审美、编码、产品 MVP 与电商判断等专项“器官”。
- 积累可复用知识、提示词和评测集。
- 为后续接入向量库、Agent、RAG、评测系统和微调小模型提供高质量语料。

## 维护原则

1. 事实、判断和假设分开记录；示例不得冒充真实经历。
2. 每条重要结论尽量附带背景、约束、依据、结果和复盘。
3. 保持目录命名稳定，优先增量补充，避免频繁搬迁。
4. 定期删除过时信息或标记其适用时间与边界。
5. JSONL 每行必须是一个独立、合法的 JSON 对象。

## 目录导航

- `00_profile/`：个人画像、能力、偏好和反模式。
- `01_projects/`：按项目沉淀上下文、决策、过程和复盘。
- `02_organs/`：专项判断能力、规则、样本与评测案例。
- `03_knowledge/`：按领域整理的可复用知识。
- `04_cases/`：跨项目的结构化案例库。
- `05_prompts/`：经过验证的提示词资产。
- `06_eval/`：评测集与期望行为。

## 后续检索与 Agent 使用

后续可按目录、标题、标签、项目、时间、适用条件等元数据切分并建立索引。Agent 或 RAG 应优先检索具体案例和规则，再参考个人偏好；评测系统使用 `06_eval/` 和各专项评测集验证回答是否符合 PersonOS。
