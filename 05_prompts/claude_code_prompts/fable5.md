# Fable5 全库架构审计 + 最大遗漏补充计划

## 元数据

- 状态：已验证
- 分类：claude_code_prompts
- 工具或场景：Claude Code 中让 Fable5 执行全库深度审计，找出最大遗漏并输出施工级计划
- 版本：v1
- 最后更新：2026-07-07
- 标签：claude-code, fable5, full-audit, architecture-review, 施工计划, 方案规划

## 用户动作/判断来源

- 用户主动问 '当前开发过程中最大的遗漏是什么，下次补充'
- Fable5 执行全库审计（架构/持久化/鉴权/密钥/错误处理/测试/文档九维度），产出 96 行结构化 plan
- 用户认可该 plan 的价值、结构和粒度，要求将其沉淀到 PersonOS

## 输入要求

- Agent 可完整读取目标代码库（grep/glob/read）
- 审计前先锁定项目定位，明确不做范围（如 '个人内部工具' vs '生产服务'）
- 审计维度覆盖：架构、持久化、鉴权计费、密钥管理、错误处理、测试、文档、依赖、构建配置

## 提示词正文

```text
当前开发过程中最大的遗漏是什么，下次补充

请做全库审计，覆盖以下维度后输出：
1. 最大遗漏（最影响使用/开发体验的问题）
2. 次要遗漏（可顺带修复的问题）
3. 明确不做项（关联其他服务/模块的责任边界）

输出要求：
- 审计前先明确项目定位，排除无关维度
- 方案选型需有技术理由和已验证的 API 契约/约束
- 每条改动精确到文件名和行号
- 施工顺序依赖驱动
- 10 条左右可执行验证步骤
- 每种风险标注优先级和处理策略
```

## 输出格式

Context（需求+审计结论+明确不做项）→ 方案选型（技术选型+已验证的API契约）→ Schema（如涉及数据层）→ 新建文件清单 → 修改文件清单（含精确文件名+行号）→ 施工顺序（依赖驱动）→ 验证步骤（约10条可执行项）→ 风险清单（按优先级排列，每项含处理策略）

## 成功标准

- 覆盖所有审计维度
- 每项改动有具体文件名和行参考（代码仓库路径）
- 施工顺序有前置依赖标记
- 风险清单有优先级和具体处理策略

## 验证结果

已验证。该 plan 已在 mono-web 项目上产出一份 96 行的完整计划，已被用户认可并待执行。

## 适用边界

- 适用于中型 Next.js / 单页 Web 项目的深度审计
- 不适用于微服务全链路审计（缺少跨服务依赖分析）
- 不适用于纯前端视觉项目（缺少视觉/动效审计维度）
- plan 的输出假设 Agent 有能力执行所有改动，不走人工 PR review 流程
