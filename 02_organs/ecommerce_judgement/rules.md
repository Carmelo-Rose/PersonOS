### 规则名称：电商自动化风控优先做闭环，不自动绕过验证

- 规则编号：待生成或待填写
- 状态：待验证
- 适用场景：做电商平台（抖店、罗盘等）数据采集或自动化，面临平台风控与账号风险时
- 正向要求：优先建立“发现-暂停-留证-人工恢复”闭环，用业务结果（采集覆盖率、成功项、失败项）判定风险，把风控做成可视化、低维护的观测体系
- 应避免：尝试自动通过或绕过平台验证、自动扩散风险；仅凭进程退出码判定成功
- 判断依据：进程正常退出不等于业务成功；自动绕过验证会推高账号封禁风险；偶尔查看的运营系统适合一张事实表加分类视图加仪表盘
- 示例：
- 正面：统一飞书“自动化风控记录”表（状态/运行/风险事件五视图），以业务结果判失败（0/20、0/5 视为严重失败），人工恢复
- 反面：把采集脚本跑通即当成功，或用独立未登录 Chrome profile 试图规避登录态拦截
- 允许例外：官方开放 API 或已授权数据源场景可简化
