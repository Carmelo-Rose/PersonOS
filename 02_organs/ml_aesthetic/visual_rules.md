### 规则名称：聊天输入框视觉对齐与比例判断规则

- 规则编号：待生成或待填写
- 状态：已验证
- 适用场景：评估或实现 AI 对话入口、聊天输入框、长文本 composer、极简 AI 首页等核心输入控件。
- 正向要求：默认态要同时满足几何中心和视觉中心；长文本展开态要采用上方文本区 + 底部工具栏的两层结构；图标尺寸、线宽、位置应参考成熟产品并保持轻量克制。
- 应避免：不要只看 CSS 盒模型中心就判定完成；不要用字体字符替代需要精确对齐的图标；不要让加号、发送按钮、模式按钮等控件在比例上压过输入框；不要把展开态仍当成单行 flex 排布。
- 判断依据：用户多次用截图和浏览器标注校准 Mono composer：视觉上偏下、偏大、突兀、展开态位置不对，即使功能可用也不能接受；最终需要按 ChatGPT/Gemini 的比例与布局语义修正。
- 示例：
- 正例：ChatGPT/Gemini 式首屏，中心问候 + 单一输入框 + 少量快捷入口，输入框默认态轻量，展开态底部工具栏清晰。
- 反例：Mono 初版 composer 中加号使用字体字符导致视觉偏下；CSS 加号线长过大导致突兀；发送按钮 42px 在窄屏输入框内显得过重。
- 边界例：几何中心线相同但字形重心偏下时，仍应按视觉效果修正。
- 允许例外：如果是无障碍大字号模式、移动端触控优先界面或明确要求高对比大按钮，可放宽按钮尺寸，但仍要保持视觉中心和层级克制。
- 置信度：高
- 用户动作/判断来源：用户在 Mono Web 原型中指出输入框和外框看起来不对称，即使代码几何中心对齐也仍要求处理视觉重心。, 用户指出字体加号偏下、CSS 加号过大、发送按钮偏大、长文本展开态加号位置不符合 GPT 原版。, 用户提供 ChatGPT/Gemini 截图作为正例，要求按成熟产品比例校准。
