## 反模式记录模板

### 反模式名称：把模糊指令丢给弱模型 / 子代理

- 适用场景：用 AI 子代理或弱模型做设计 / 开发
- 典型表现：只给风格名或模糊目标 → 产出换皮模板、被评“都是垃圾”；弱模型直接开发结果不可控（“一坨屎”）
- 为什么有害：返工成本高，浪费 token 与时间；且执行体容易交付前不自评
- 早期信号：指令里只有抽象风格词、没有具体参考 / 约束；交付前未做质量自评
- 替代做法：先给具体参考（URL / 截图）+ 从身份 / 概念出发 + 写明约束；强模型定架构、先写 SPEC 再落地
- 允许例外：纯探底 / 试错（已明确预期效果不佳，仅用于摸边界）
