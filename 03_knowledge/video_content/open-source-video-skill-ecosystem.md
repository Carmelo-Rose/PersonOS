## 用户动作/判断来源

- 用户询问是否存在做视频的 skill 开源社区
- 本次会话中对本机 skill 目录和相关开源项目的扫描与对照

## 适用条件

- 用户在找可复用的视频创作 skill、agent workflow 或视频生产工具链
- 需要判断某个 video skill 是否属于成熟统一社区，还是只是局部仓库或单点工具

## 核心内容

以后找视频相关的可复用能力时，不要默认存在一个统一的『视频 skill 社区』。更稳妥的检索顺序是：先看本地已安装的 skill 目录，再看通用 agent skills 汇总，再看视频生产工具链，最后看视频生成/编辑项目。把它们按能力层拆开比按社区名称找更可靠。

## 边界与例外

- 如果用户明确指定某个社区、仓库或插件名，就直接按指定目标核查，不要再泛化。
- 这个判断只描述当前生态检索策略，不代表所有仓库都不可协作或不可组合。

## 失效风险

开源项目活跃度、仓库归属和工具链名称变化很快，需要定期复核最新仓库状态与社区活跃度。
