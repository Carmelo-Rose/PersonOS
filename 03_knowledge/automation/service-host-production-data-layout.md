## 适用条件

- 项目以 Windows 服务机作为运行环境
- 项目同时包含结构化业务数据和图片、视频、生成结果等二进制对象
- 需要多项目共存、统一启停、备份和故障定位

## 核心内容

- 源码统一放在 D:\hyk_sort\apps；平台程序放在 D:\hyk_sort\platform；唯一运行数据根目录为 D:\hyk_sort\prod-data；运维脚本放在 D:\hyk_sort\ops；凭据单独放在受限的 D:\hyk_sort\secrets。
- PostgreSQL 保存结构化业务数据、状态、哈希、对象 key 和任务元数据；每个项目使用独立数据库和最小权限账号。
- MinIO 保存图片、视频、模型结果等二进制对象；应用只持久化对象 key，不把二进制塞进 SQL，也不把运行数据散落在源码目录。
- 日志、运行时缓存和兼容旧程序的数据通过项目 ID 归档到 prod-data；迁移前先做可回滚备份，迁移后用健康检查、数据库表检查、对象存储读写和备份检查验收。

## 边界与例外

- 数据库不应成为图片视频的长期二进制仓库；只有很小且强事务性的内容才考虑直接存数据库。
- 源码目录中的 data/logs/output 可能是兼容旧代码的映射/Junction，不能仅按目录名判断为重复数据。
- 单机同盘备份不能等同于容灾；不可丢失数据还需要另一块物理磁盘、NAS 或异地对象存储。
- 涉及浏览器 profile、采集任务或 GPU 服务时，迁移前必须单独确认进程停止、独占约束和恢复方式。

## 失效风险

中：Windows 单机 MinIO 适合当前基线，但正式高可用生产应迁移到多盘/Linux/多节点，并把备份复制到另一物理位置。
