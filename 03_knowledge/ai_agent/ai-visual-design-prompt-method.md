## 用户动作/判断来源

- 用户让 ChatGPT 分析 activetheory.net 与当前设计的差距
- ChatGPT 首次回复指出核心问题：缺少沉浸式 WebGL、空间感、镜头感、交互感
- ChatGPT 第二次回复生成 182 行详细迭代 prompt，按场景/粒子/镜头/页面/交互/技术六个维度拆解
- 用户对比直接给 Claude Design 描述风格 vs 用 ChatGPT 转译后的详细 prompt，确认后者效果明显更好

## 适用条件

- 需要让 AI 代码 Agent 实现特定视觉风格时
- 目标风格有明确参考网站或作品时
- 直接描述风格关键词效果不佳时
- 需要将模糊的审美偏好转化为可执行的技术指令时

## 核心内容

当目标是有明确参考的视觉风格时，采用三步法：(1) 让分析型 AI（如 ChatGPT）对比参考网站与当前设计，指出具体差距；(2) 让同一 AI 生成详细的、按维度拆解的执行 prompt（包含结构、交互、技术约束、避免事项）；(3) 将详细 prompt 喂给代码 Agent（如 Claude Design / Codex）。关键差异：不是告诉 Agent '我要 Active Theory 风格'，而是拆解成具体的技术要求（WebGL 粒子系统、鼠标视差、镜头推进、空间层次等）。prompt 越接近技术规格书，Agent 产出越接近预期。

## 边界与例外

- 仅适用于有明确参考网站/作品的场景
- 对于无参考的纯原创设计，需要先通过 moodboard 确定方向再使用此方法
- prompt 长度需要平衡：太短则 Agent 自由发挥偏离方向，太长则 Agent 可能忽略部分要求

## 失效风险

方法本身稳定，但具体 prompt 模板需要随技术栈和参考风格变化而更新
