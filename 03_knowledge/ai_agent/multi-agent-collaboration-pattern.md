# 多 AI Agent 分工协作模式：大脑 + 执行者

## 用户动作/判断来源

- 用户在 Portfolio 项目中实践了多 Agent 分工：ChatGPT 做设计分析、Claude Design 做首轮实现、Mimo 做探底实验
- 用户发现 Claude Design token 成本过高后，提出 Claude Cowork（Opus4.8 大脑）+ Claude Code（Mimo 执行）的新链路
- 用户明确认知：Mimo 直接开发大概率效果不好，需要强模型做决策层

## 适用条件

- 项目需要同时具备高质量决策和大量执行工作时
- 单一高成本 Agent 的 token 消耗不可持续时
- 可以将决策/架构与设计/执行明确分层时

## 核心内容

多 AI Agent 协作的核心模式：(1) 大脑层 — 使用最强模型（如 Opus4.8）负责架构决策、任务拆解、质量把关；(2) 执行层 — 使用低成本模型（如 Mimo）负责代码编写、文件操作、构建部署等具体执行；(3) 顾问层 — 使用分析型 AI（如 ChatGPT）做设计分析、prompt 工程、方案对比。关键原则：执行层不能独立决策，必须由大脑层拆解任务后分发；顾问层的产出需要人工或大脑层转译后才能喂给执行层。桥接方式：Claude Cowork 提供内置 API Key 桥接 Claude Code，避免额外 token 成本。

## 边界与例外

- 大脑层和执行层之间的任务拆解需要足够清晰，否则执行层会误解或遗漏
- 复杂交互逻辑（如 WebGL shader 调试）可能仍需要高能力模型直接处理
- 桥接方案的稳定性取决于平台支持，可能随产品更新而变化

## 失效风险

AI 模型定价和能力快速变化，具体工具组合可能很快过时，但分层思路稳定
